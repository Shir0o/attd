# Changelog

All notable changes to this project will be documented in this file. This changelog tracks code changes at the merge/PR level rather than project releases, providing a lightweight history to help prevent regressions.

---

## [Unreleased]

- **Background Auto-Sync for Google Drive Backup** (issue #93): Implemented periodic background synchronization to Google Drive using Workmanager (`BackgroundSyncService`). Automatically performs background sync every 12 hours when enabled, with support for network constraints (Require Wi-Fi toggle in Settings). Tracked last auto-sync execution timestamp and status in `SharedPreferences` and surfaced status in `SettingsPage`. Added comprehensive unit and widget tests.

- **OpenWiki Pages Fixed with Required YAML Front Matter**: Fixed the scheduled `OpenWiki Update` workflow still failing after the provider switch — `openwiki` (now v0.2.0) rebuilds directory indexes after each run and hard-fails if any Markdown file under `openwiki/` lacks OKF-compliant YAML front matter (a `type` field, at minimum). The 7 wiki pages committed by the initial `openwiki --init` run predate that requirement and had no front matter at all, so every scheduled run crashed with exit code 1 regardless of credentials. Added `type`/`title`/`description` front matter to `openwiki/quickstart.md`, `architecture.md`, `design.md`, `features.md`, `integrations.md`, `operations.md`, and `testing.md` (verified against the installed package's actual validation logic).
- **OpenWiki Workflow Switched to Anthropic Provider**: Fixed the scheduled `OpenWiki Update` workflow failing on its first run (missing `OPENROUTER_API_KEY` secret) by switching it to the Anthropic provider (`OPENWIKI_PROVIDER: anthropic`, `claude-sonnet-5`) using the newly added `ANTHROPIC_API_KEY` repo secret, instead of OpenRouter.
- **OpenWiki Integration & Agent Files Now Tracked**: Added the `openwiki/` generated documentation directory and `.github/workflows/openwiki-update.yml` scheduled workflow, which keeps repo docs current and opens a PR with updates. Reversed the earlier "Local-Only Agent Files" decision: removed `CLAUDE.md` from `.gitignore` and committed both it and `AGENTS.md` (kept identical, each combining the existing behavioral guidelines with an OpenWiki pointer section) so the CI workflow can update and commit them.
- **Test Coverage Threshold Elevation**: Raised the test coverage guard threshold from 94.0% to 95.0% in `tool/check_coverage.dart`. Added missing unit tests for the Google Sign-In service (`google_sign_in_service_test.dart`), and expanded coverage for domain entity copy/serialization logic and local event repository exception handling, bringing overall line coverage to over 95%.
- **PR-Agent Configuration Update**: Updated the PR Agent GitHub Actions workflow to conform with `cisa-campus-work-traker`. This switches the primary model to `gemini-2.5-flash` (with a `gemini-2.5-flash-lite` fallback) and changes the automatic trigger behavior to `/improve` instead of `/review`.
- **Redesigned Manage Backup Data Page ("Storage inspector")**: Fully redesigned the backup management screen to conform to the "BackupDataScreen" design, featuring high-quality serif typeface headers, Geist monospace key-value formatting for detailed views, and interactive table filter chips (All, Events, Sessions, Members, Families, Photos, Attendance). Mapped raw JSON database entities to a unified `DbRecord` interface with status badges (`hidden` for soft-deleted entries, `orphan` for unlinked/invalid dependencies). Added a sticky bulk-cleanup bar at the bottom that enables permanent cleanup of all flagged records via a modal confirmation sheet, alongside individual record detail drawer expansion, warning alerts, and ID clipboard-copy actions. Added raw fetching/saving methods to `LocalJsonAttendanceRepository`, `LocalJsonEventRepository`, and `LocalJsonSessionRepository` without changing the abstract interface contracts. Updated settings page and backup data page test suites to match the redesigned layout and cleanup logic, achieving 100% test coverage and static analysis compliance. Fixed a horizontal layout overflow in the record counters row to support narrow mobile screens, and wrapped the skeleton loader's shimmer chips row in a horizontal scroll view to prevent layout overflows during loading states. Refactored page layout to use a sticky header column and scrollable body list, preventing TextField disposal during scrolling. Updated the settings robot and data integrity integration tests to align with the new Storage Inspector page layout, immediate deletion flow, and correct database record counts, including adding robust search focusing, scoped row-finders to prevent TextField text collisions, and correct database assertion counts.
- **Version History & Force Sync Overhaul**: Redesigned the Cloud Version History page (`CloudBackupPage`) to match the editorial **Convocation** design system. Replaced the old list view with a detailed timeline snapshot list showing dynamic relative timestamps, formatted backup sizes, device/user labels, and soft primary change tags. Integrated the two force-sync actions ("Overwrite local" and "Overwrite cloud") as side-by-side card buttons directly on this page, and removed them from the main settings page. Updated and migrated all associated widget tests.
- **iOS Swift PM Dependencies Resolution**: Resolved and upgraded Swift Package Manager dependencies (`AppAuth-iOS` to `2.1.0` and `GoogleSignIn-iOS` to `9.2.0`) in `Package.resolved` files, ensuring consistent package versions across environments.
- **Database Corruption Recovery & Sync Resilience**: Implemented automatic local database healing by restoring data from `.bak` backup files if the main `.json` database files are corrupted or missing (in session, family, and event repositories). Resolved a file rename conflict in `DriveService` local file merge/update by rotating the old file to `.bak` first, preventing JSON corruption on Android/iOS filesystems during sync. Wrapped Google Sheets sync payload generation in a try-catch block to handle corrupted session files gracefully without crashing page loads or sync flows.
- **New-Event Roster Member Picker**: The "Roster" row on the new/edit-event form is now tappable and opens the member edit page as a non-persisting picker (`MembersPage` selection mode), returning the chosen members so they're saved only when the event is created/saved. New events default to no member pre-selected (an empty roster still means "All members" downstream, per the existing convention). Editing an event pre-checks its current members. `AddEventPage` now takes an `attendanceRepository`.
- **Onboarding "07 · Onboarding" Conformance**: Brought the onboarding flow into line with the final design bundle. The "Quick marking" deck art is now a balanced, centered two-card fan (Jane behind-left, John front-right) with the Present/Absent stamps anchored to each card's own corner instead of floating at arbitrary offsets. Slide 04 copy corrected from "Local-first. Encrypted backup." to "Local-first. Always yours." to drop the false encryption claim (storage is local-first with sync to the user's own Google Drive).
- **Insights Follow-ups: 80% Threshold, Tappable Trends Rows & Absent-Count Fix**: (1) The Regulars page now uses a true ≥80% rule, computed as a fraction of the window (`kPatternThreshold = 0.8`) instead of a fixed 7-of-8, so the helper text reads "80%+" (not the old computed "88%+") and short windows qualify adaptively; full 8-session behavior is unchanged. (2) Recent-session rows on the Trends page are now tappable and open that session's summary (repos threaded through `EventTrendPage` and `_ConsistentTrendStrip`). (3) Fixed Trends present/absent counts that showed e.g. "5 present / 0 absent" when the summary correctly showed "5 / 11" — Trends derived counts from raw `session.records` (which omit absentees), and now computes via `SessionRoster` over the event roster, matching the Session Summary.
- **"05 Insights" Regulars & Trends Rebuild**: Brought the `ConsistentMembersPage` ("The reliable few") and `EventTrendPage` ("Trends") screens up to the `insights.jsx` "05 Insights" design. Regulars now scrolls and adds a two-tile count strip (Regulars `n / total` · Avg attendance %), a richer hero card ("Most consistent" eyebrow, larger name, full-width session ribbon with "N weeks ago → This week" labels), an "Also reliable" ranked list where each row carries a percentage + `hits/window` ratio beside its mini ribbon, and a "Lives on your device · never shared" footer. Trends was reworked from a present/absent hero + sparkline into the editorial layout: a 72px average-rate hero with an "↑ up from X% prior" delta, a functional 12 wk / 6 mo / Year range selector (switching the window among 12/26/52 sessions), a bar chart with a dashed AVG line inside a soft card, Best/Lowest/Average stat tiles, a "Recent sessions" list (date · weekday · time, present·absent counts, %, chevron), and a "100% local · export to CSV anytime" footer. The Regulars link card was removed from the Trends body (Regulars stays reachable from the session summary). Tests updated in lockstep.
- **Session Summary Editorial Header**: Reworked the session summary header to match the "05 Session Summary" design — the session title moves out of the centered AppBar into the page body as a large serif display title, preceded by a primary-tinted `SAVED · <time>` eyebrow (the save timestamp), replacing the plain "Session Date: …" line. The hero present/absent stats, Regulars/Trends strip, and roster are unchanged.
- **Hub Upcoming Events Chronological Sorting**: Events in the "This week"/"Upcoming" section now sort by next occurrence date first, then by time-of-day, instead of by time-of-day alone — so an earlier-in-the-week event always appears above a later one regardless of clock time. Reuses the existing `getNextOccurrence` helper.
- **PR-Agent (Qodo) Automated PR Review**: Added a self-hosted `pr_agent.yml` GitHub Actions workflow running Qodo's open-source PR-Agent on each PR. Uses Gemini (`gemini/gemini-2.5-pro`, falling back to `gemini-2.5-flash`) via a `GEMINI_API_KEY` repo secret, with auto `/review` enabled (auto `/describe` and `/improve` off; both still available on-demand via PR comments).
- **Hub "Nothing scheduled" Rest State**: When no events are scheduled today but more are still coming this week, the hub now shows a calm, non-actionable rest-state card ("Nothing scheduled.", "A quiet day — no attendance to take. Your next event is in N days.") with a tap-through "Next up" affordance to the soonest upcoming event, matching the `HubNoneToday` design. The following section is labelled "This week" instead of "Upcoming". Previously this case silently dropped straight into the upcoming list with no hero.
- **Smart Defaults in "Mark everyone" Sheet**: The bulk "All" action sheet now offers a third **Smart defaults** option alongside All present / All absent, matching the "02 Quick Marking" design. Picking it resolves each member from recent attendance history (present if here ≥80% of the last 8 sessions, absent if ≤20%, otherwise left untouched) via the existing `resolveDefault` rule, reusing a new `applyBulkSmartRecords` helper. Wired into both the in-session deck list view and the session summary, with an undo snackbar reporting how many members the smart guess resolved.
- **Hub "Taken" Card Routing Fix**: Tapping a marked ("Taken") event card on the hub now always opens the session summary instead of reopening the marking deck. The tap handler previously routed to the deck whenever `records.length < memberIds.length` (e.g. after a member was added post-marking), which contradicted the "Taken" badge; navigation now matches the badge — any existing/confirmed session opens its summary, where individual statuses can still be edited.
- **Hub "Taken" State After All-Present Confirm**: A session confirmed with "Start with all present" (no exceptions toggled) is no longer silently discarded on return to the hub, so the event correctly shows its taken/Marked state. Keep-vs-discard is now decided by the action taken — confirming (Done) keeps the session; cancelling (X / system back) on a freshly created session discards it if untouched, or prompts "Save as is / Discard" once the user has marked anyone — via a `PopScope` on the deck gated by a `deleteOnCancel` flag (only set for freshly created sessions, so resuming an existing incomplete session and backing out preserves its records), instead of guessing from whether records still bore the preseed author (`System (Preseed)`), which misclassified an untouched all-present confirmation as an abandoned session.
- **Hub Event Row "Taken" Status & Three-Dot Menu**: Moved the marked status on hub event rows ("Also today" and "Upcoming"/"This week") from the right-side `Taken` pill into the row subtitle (a check icon + "Marked" tinted with the present color), and added the same three-dot overflow menu (Manage Members / View History / Edit / Delete) the hero card uses to the right edge of each row, matching the "01 Hub" design.
- **Roster List Duplicate-Row Fix**: Deduplicated members by id when building the deck list view (both By status and By family groupings), so a member that belongs to more than one family (e.g. a real family plus an auto-singleton) renders only once. Previously such members appeared as two identical rows whose toggles moved together. Also scoped attendance record overwrites to match real members by id (never by name), so toggling one member no longer wipes another member that happens to share the same display name. The historical-session "Edit Member" dialog now notes that a name correction applies to that report only (not the global roster or other sessions).
- **Quick Marking Deck & Confirm List Redesign** (#109): Rebuilt the quick-marking deck and roster confirm list to match the "02 Quick Marking" design — centered Deck/List toggle and contained progress bar, a resized undo/absent/present action row, a quiet ghost "Add guest" button, drag hint zones and a next-card peek, and a fixed singleton caption ("Loner" / "{family} family" instead of the repeated name). Moved roster grouping (By status / By family) out of the in-session toggle into a per-event preset that is asked once and inherited thereafter.
- **Quick Marking Start-Mode Routing** (#107): The start mode now picks the entry view — All absent → speed-swipe deck, All present / Smart → pre-marked roster List view — via a new `initialListMode` param, so presets no longer force the user to swipe through everyone.
- **Smooth Footer-Button Swipe** (#106): The Present/Absent footer buttons now play the same fly-off animation as a manual swipe via a new `SwipeableCardController`, with a re-entry guard against double-advance. Removed the artificial 800ms deck/summary skeletons and fixed an undo blank-card reuse bug.
- **Hub Today Highlight & "Also today" Group** (#105): The hero highlight card is reserved for the soonest unmarked today event (one at a time); remaining same-day events drop into a new "Also today" group, non-today events move under "This week", and the header gains an "{n} EVENTS" pill when more than one event falls on today.
- **Hub Convocation "01 Hub" Rebuild** (#104): Rebuilt the Hub on the Convocation design system — an "Up next" hero card (TODAY·time pulse pill, large serif title, day-chip row, Expected / Last week stats, Start button or Taken pill) and compact "Upcoming" rows, reusing the existing Conv* primitives.
- **Onboarding Convocation Rebuild** (#102): Rebuilt the onboarding flow on the Convocation design system with four bespoke editorial art widgets (replacing the five MockAttendance* widgets), collapsed 5 slides to 4, and preserved the completion/skip navigation logic.
- **Crashlytics Build Phase Automation**: Integrated `[Firebase Crashlytics] Upload Symbols` build phase into the Xcode project file to automatically generate and upload dSYMs to Firebase Crashlytics on builds/archives, using the SPM Crashlytics run script.
- **Settings Page Convocation Overhaul** (issue #84): Rebuilt `settings_page.dart` with the Convocation design system — a Fraunces 32 "Settings" display title, eyebrow section headers (`ConvEyebrow`), a Google Drive hero card (`ConvCard`) with primary-tinted icon, email subtitle, green status dot, and side-by-side Sign-out / Sync buttons, and each setting row as a `ConvCardSoft` with primary-tinted leading icon plus trailing chevron/value/toggle. Restyled the 800ms loading skeleton and footer (`Attendance · Version 1.2.0+13`) to match. All existing sections, controllers, stream subscriptions, and the `_performOperation`/`_dataModified` flow are preserved; tests updated in lockstep.
- **Equal Height Regulars & Trends Cards**: Aligned the heights of the "Regulars" and "Trends" dashboard cards inside the attendance session summary page by wrapping the row in `IntrinsicHeight` and applying `CrossAxisAlignment.stretch`. Simplified the card content by removing the redundant "Consistent members" and "See sparkline" subtext.
- **Smart Preseeds Skipping & Deck Undo Stack**: Modified the Attendance Deck page to respect and skip members pre-marked by "Smart defaults (from past 8 sessions)" (recorded under a distinct `System (Preseed - Smart)` actor name). Implemented an O(1) set-based caching mechanism to eliminate UI-thread lookups overhead, and introduced a list-based `_history` index stack to replace standard index-decrementing, ensuring robust and correct Undo navigation. Added comprehensive widget testing.
- **Changelog & Agent Workflows**: Established the `CHANGELOG.md` to track git merges/commits, updated instruction systems (`CLAUDE.md`, `GEMINI.md`) to require pre-change context checking and post-change updates, and cleaned up redundant instruction files (`AGENTS.md`).
- **Local-Only Agent Files**: Ignored `CLAUDE.md` and `GEMINI.md` in `.gitignore` and untracked them from git repository to ensure agent instruction files remain local-only.
